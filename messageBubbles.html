Sprout's message bubble is a complex component that is capable of displaying any message type, from any social network, in addition to controls that power Sprout tools, such as tasking, tagging, or analytics.

Message Bubbles Message Analytics
The Problem

Create a clean, familiar message bubble that provides users with all the information they need to do their jobs successfully. This includes incoming and outgoing message features for all the major network message types. In turn, the message needs to expose features and value that Sprout offers in addition to what the networks provide. All this in a tidy little package...

The Work

Creating the Sprout message bubble began with research. Auditing the messages already in existence in Sprout, auditing works in progress, and finally auditing the networks (FB, IG, LI, TW, G+) for anything else. Next, I did some competitive analysis by examining the network message bubble designs looking for inspiration on pieces of experiences that could scale with our needs while contributing to the "familiar" experience.

With some great research in hand it was straight to sketch to start realizing pieces of the experience in context. Along the way, I was able to vet static designs through prototyping to ensure the static interactions behaved as I intended.

Finally, I user tested the bubbles with internal users and real customers. This began the evolution of the bubble through constant iteration that continues to this day. In order to manage that evolution, I created a thorough archive of documentation, symbols, and prototypes while our front-end devs created the atomic elements for our React library.
